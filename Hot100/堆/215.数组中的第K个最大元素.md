### 题目描述
给定整数数组 `nums` 和整数 `k`，请返回数组中第 `**k**` 个最大的元素。

请注意，你需要找的是数组排序后的第 `k` 个最大的元素，而不是第 `k` 个不同的元素。

你必须设计并实现时间复杂度为 `O(n)` 的算法解决此问题。


### 思路
如果排序的话就很容易，但是时间复杂度是O(nlogn)
在一个升序数组中，第K大的数的索引是n-k（第一大的是n-1...）
因此我们可以随机找一个数字，将数组分为两部分，左边小于他，右边大于他，这样判断他的位置是否是n-k，是则返回，不是则在左右两部分重复上述操作

**但是其实没有用到堆




### 代码
```python
class Solution:

    def tougharr(self,nums,left,right):

        i=randint(left,right)
        tem=nums[i]
        
        nums[i],nums[left]=nums[left],nums[i]

        i=left+1
        j=right

        while True:

            while i<=j and nums[i]<tem:
                i+=1
            while i<=j and nums[j]>tem:
                j-=1

            if i>=j:
                break

            nums[i],nums[j]=nums[j],nums[i]
            i+=1
            j-=1

        nums[left],nums[j]=nums[j],nums[left]
        return j

    def findKthLargest(self, nums: List[int], k: int) -> int:

        n=len(nums)
        target_index=n-k
        left=0
        right=n-1

        while True:
        
            i=self.tougharr(nums,left,right)

            if i ==target_index:
                return nums[i]
            if i>target_index:
                right=i-1
            else:
                left=i+1
```
